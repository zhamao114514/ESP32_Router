# ESP32-S3 校园网共享热点
 ESP32-S3 实现校园网多设备共享上网，使用MAC克隆 + 自动Portal认证。
 ## 功能
 - STA连接校园网，AP开启本地热点
 - MAC伪装绕过单设备登录限制
 - 自动网页认证、掉线重连、心跳保活
 - NAT网络转发，多设备共用网络
 ## 环境要求
 - 硬件：ESP32-S3
 - 编译环境：Arduino IDE，ESP32 核心 ≥ 2.0.14
 - 配置：开启 PSRAM、IPv4+IPv6 Full LWIP、Huge APP 分区
 ## 快速配置
 修改代码顶部 `用户配置区`：
 1. 校园网WiFi名称、密码
 2. 认证服务器IP、端口
 获取方法:在校园网登录页面按下F12点击网络,按F5刷新一下
 点击右侧新出现的文字找到请求 URL,通常第一段就含有ip+端口
 分别填写好后再复制完整的地址举例:http://10.200.1.11:801/eportal
 替换第146行的地址 
 4. 上网账号、密码
 5. 自建热点名称、密码
 6. 已认证设备的真实MAC地址
 找一个手机关闭随机MAC,正常连接校园网登录,然后在手机设置 → 关于本机中找到手机MAC填上就好
 ## 使用
 1. 配置参数 → 验证编译,显示编译完成 → 上传程序的时候按住"BOOT"再点按"RST"
 2. 串口监视器 115200 波特率查看状态
 3. 连接设备热点即可上网
 ## 注意事项
 1. 禁止公开明文账号、密码、MAC地址。
 2. 关闭手机/电脑WiFi随机MAC功能。
 3. 供电不足会导致WiFi频繁断连，请使用稳定电源。
 ## 常见问题排查:
 1.  Q: 串口输出乱码？
 A: 检查串口监视器波特率是否为 115200
 2.  Q: 连接校园网失败？
 A: 检查 CAMPUS_SSID 和 CAMPUS_PASS 是否正确
 3.  Q: 能连 WiFi 但认证失败？
 A: 先确认 Portal 服务器地址/端口是否正确
 方法：在电脑浏览器打开 http://你填的IP:你填的端口
 如果能打开登录页面，说明地址对了
 否则需要修改 AUTH_HOST_IP 和 AUTH_PORT_NUM
 4.  Q: "账号或密码错误"？
 A: 确认 USER_ACCOUNT 和 USER_PASSWORD 是否和
 手动在浏览器登录时用的一致。先手动登录一次排除账号问题
 5.  Q: 认证成功但热点不能上网？
 A: 检查 NAT 日志（串口输出看有没有"✅ NAT转发已启用"）
 如果显示超时，先忽略，等心跳检测看看能不能通
 6.  Q: 连上热点但设备显示"无互联网连接或自动跳转到校园网登录界面"？
 A: 可能是MAC克隆出现问题
 拿被克隆MAC的手机连接热点正常登录一次就好（或许呢？反正我这样是可以的）
